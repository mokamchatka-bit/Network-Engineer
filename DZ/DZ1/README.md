# Лабораторная работа: Базовая настройка коммутатора


## Описание
Лабораторная работа посвящена основам настройки коммутатора Cisco 2960. В ходе работы вы научитесь:
- Проверять заводские настройки коммутатора
- Настраивать базовые параметры устройства (имя, пароли, баннеры)
- Назначать IP-адрес для удалённого управления через SVI (VLAN 1)
- Настраивать доступ через консоль и Telnet
- Проверять связность сети с помощью утилиты ping
- Удалённо управлять коммутатором через Telnet

## Необходимое оборудование и ПО
- 1 коммутатор Cisco 2960 (Cisco IOS 15.2(2) lanbasek9 или аналогичный)
- 1 ПК с Windows
- Программа эмуляции терминала (Tera Term, PuTTY)
- Консольный кабель (RJ-45 – DB9/DB25/USB)
- Кабель Ethernet (витая пара)

## Части лабораторной работы
1. **Проверка конфигурации коммутатора по умолчанию**
2. **Настройка базовых параметров устройства**
3. **Проверка сетевых подключений**

## Основные шаги настройки
### Часть 1: Проверка настроек по умолчанию
- Подключение к коммутатору через консоль
- Проверка текущей конфигурации (`show running-config`)
- Проверка интерфейсов (`show interface`)
- Проверка VLAN 1 (`show interface vlan 1`)
- Проверка версии IOS (`show version`)

### Часть 2: Базовая настройка
#### Настройка коммутатора S1:
enable
configure terminal
no ip domain-lookup
hostname S1
service password-encryption
enable secret class
banner motd # Unauthorized access is strictly prohibited. #
interface vlan 1
ip address 192.168.1.2 255.255.255.0
no shutdown
exit
ip default-gateway 192.168.1.1
line console 0
password cisco
logging synchronous
login
exit
line vty 0 15
password cisco
login
exit
end
write memory

text

#### Настройка ПК (PC-A):
- IP-адрес: 192.168.1.10
- Маска: 255.255.255.0
- Шлюз по умолчанию: 192.168.1.1

### Часть 3: Проверка подключений
- Проверка связи с самого на себя (`ping 192.168.1.10`)
- Проверка связи с коммутатором (`ping 192.168.1.2`)
- Подключение к коммутатору через Telnet
- Сохранение конфигурации

## Ключевые команды
```bash
show running-config      # Показать текущую конфигурацию
show interfaces          # Показать состояние интерфейсов
show interface vlan 1    # Показать параметры VLAN 1
show version            # Показать версию IOS
ping <IP-адрес>         # Проверить связность
telnet <IP-адрес>       # Удалённое подключение через Telnet

Приложение А: Сброс коммутатора к заводским настройкам
bash
enable
delete vlan.dat
erase startup-config
reload
# При запросе о сохранении конфигурации ответить "no"
# При запросе о входе в диалог начальной настройки ответить "no"
Примечания
По умолчанию VLAN 1 уже существует на коммутаторе и используется для управления

Telnet не является безопасным протоколом (данные передаются открыто)

Для шифрования паролей используется команда service password-encryption

В реальных сетях рекомендуется использовать SSH вместо Telnet
